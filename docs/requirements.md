# 結び日 — ふたりの入籍日を、一緒に探そう。
# 要件定義・技術選定

## 1. プロジェクト概要

### アプリ名
**結び日（むすびび）**
> ふたりの入籍日を、一緒に探そう。

### コンセプト
期間を指定すると、縁起の良さをスコアリングした候補日ランキングを返してくれるWebアプリ。
六曜・吉日・愛にまつわる記念日・語呂合わせ・数字の並びなど複数の観点で日付を評価し、入籍に最適な日を提案する。

### 開発体制
- 個人開発（1人）

---

## 2. 機能要件

### MVP（必須機能）

- 期間指定（開始日〜終了日）で候補日を絞り込める
- 各日付を複数条件でスコアリングし、ランキング形式で表示する
- 各日付に「なぜおすすめか」の理由タグを表示する（例：大安・一粒万倍日・ゾロ目）
- スコアの高い順に並べ替えができる
- スマホでも快適に使えるレスポンシブ対応

### v2候補（拡張機能）

- ふたりの誕生日を入力すると、それにまつわる日付も加味できる
- 条件のON/OFFフィルター（例：「六曜を重視したい」「語呂合わせは不要」など）
- URLコピーによる結果シェア機能
- 凶日フラグ表示：不成就日・黒日などに該当する場合、ランキング上に「⚠️ 避けた方が良い日」として警告を表示する

---

## 3. 非機能要件

- バックエンドレス構成（サーバー費用ゼロ）
- 全ての計算・ロジックはフロントエンドで完結
- 外部APIのレスポンスはフロントでキャッシュし、無駄なリクエストを避ける

---

## 4. ソースコード管理・セキュリティ

### リポジトリ
- GitHub の**公開リポジトリ**で管理する
- Issues・Projects・GitHub Actions などの GitHub 機能を積極活用する

### セキュリティ方針

公開リポジトリのため、機密情報のコード混入には特に注意する。

| 対策 | 内容 |
|---|---|
| `.env` ファイルの除外 | APIキーなどの機密情報は `.env` に記載し、`.gitignore` に必ず追加する |
| `.env.example` の提供 | 変数名だけを記載したサンプルファイルをコミットし、他の開発者が参照できるようにする |
| GitHub Secret scanning の有効化 | 誤ってキーをコミットした場合に自動検知できるようにする |
| Vite の環境変数の仕様を把握 | `VITE_` プレフィックスの変数のみブラウザに公開される仕様を理解した上で命名する |
| Vercel への環境変数登録 | デプロイ時の機密情報は Vercel の環境変数設定画面から登録し、コードには含めない |

> 💡 **現時点では**暦APIはキー不要の無料APIのため機密情報は発生しないが、将来的に認証が必要なAPIを追加する際に備えて、開発当初からこの方針を徹底する。

## 5. スコアリング設計

複数条件の加算方式を採用。条件が重なるほど高スコアになる。

### APIから取得して判定するもの

| 条件 | 判定フィールド | 加点 |
|---|---|---|
| 大安 | `rokuyou === "大安"` | +30 |
| 友引 | `rokuyou === "友引"` | +15 |
| 天赦日 | `tensyabiflg === true` | +25 |
| 一粒万倍日 | `hitotubuflg === true` | +20 |
| 大明日 | `daimyoubiflg === true` | +10 |
| 寅の日 | `zyunisi === "寅"` | +10 |
| 巳の日 | `zyunisi === "巳"` | +10 |
| 国民の祝日 | `holiday !== ""` | +15 |

### フロントエンドで自前判定するもの

| 条件 | 判定ロジック | 加点 |
|---|---|---|
| いい夫婦の日 | 月 === 11 && 日 === 22 | +25 |
| ダーズンローズデー | 月 === 12 && 日 === 12 | +25 |
| バレンタインデー | 月 === 2 && 日 === 14 | +20 |
| ホワイトデー | 月 === 3 && 日 === 14 | +20 |
| 七夕 | 月 === 7 && 日 === 7 | +20 |
| 恋人の日 | 月 === 6 && 日 === 12 | +20 |
| プロポーズの日 | 月 === 6 && 日 === 1 | +15 |
| ゾロ目 | 月 === 日（例：3/3, 11/11） | +15 |
| ジューンブライド | 月 === 6 | +5 |
| 連番 | 日 === 月 + 1 など | +10 |


### 凶日（マイナス補正・警告フラグ）

吉日と凶日が重なった場合、スコアを減点しつつ警告タグを表示する。
なお、天赦日は凶を打ち消す力があるとされるため、凶日と重なっても減点しない特例とする。

| 条件 | 判定ロジック | 補正 | 備考 |
|---|---|---|---|
| 仏滅 | `rokuyou === "仏滅"` | -20 | 六曜最凶日 |
| 赤口 | `rokuyou === "赤口"` | -10 | 正午以外は凶 |
| 不成就日 | フロント自前計算（約60日周期） | -15 | 始めたことが実らない日 |
| 黒日（受死日） | フロント自前計算 | -20 | 葬儀以外は全て凶とされる最凶日 |

> ⚠️ **実装上の注意**：不成就日・黒日はAPIから取得できないため、フロントで独自計算が必要。計算アルゴリズムは実装フェーズで別途調査・検証する。
### レア度表示（スコア → ★変換）

スコアに応じてランキング上で星を視覚表示する。

| スコア | レア度 |
|---|---|
| 70点以上 | ★★★★★ |
| 50〜69点 | ★★★★☆ |
| 35〜49点 | ★★★☆☆ |
| 20〜34点 | ★★☆☆☆ |
| 20点未満 | ★☆☆☆☆ |

## 6. 外部API

### 暦API（koyomi.zingsystem.com）

| 項目 | 内容 |
|---|---|
| URL | `https://koyomi.zingsystem.com/api/` |
| 提供形式 | JSON |
| 料金 | 無料 |
| 運営 | 個人（レンタルサーバー） |

**リクエスト例（月単位）**

```
GET https://koyomi.zingsystem.com/api/?mode=m&cnt=1&targetyyyy=2025&targetmm=06
```

**主なレスポンスフィールド**

| フィールド | 内容 |
|---|---|
| `rokuyou` | 六曜（大安・友引・先勝・先負・赤口・仏滅） |
| `hitotubuflg` | 一粒万倍日フラグ（true/false） |
| `tensyabiflg` | 天赦日フラグ（true/false） |
| `daimyoubiflg` | 大明日フラグ（true/false） |
| `holiday` | 祝日名 |
| `zyunisi` | 十二支（寅の日・巳の日の判定に使用） |

**注意事項**
- 個人運営のレンタルサーバーのため、大量リクエスト禁止
- 月単位（`mode=m`）で取得し、フロントでキャッシュして1ヶ月1リクエストに抑える

## 7. データ取得フロー

```
1. ユーザーが期間を指定（例：2025/04/01 〜 2025/09/30）
2. 対象月を洗い出す（4月〜9月 → 6ヶ月分）
3. 各月ごとにAPIを1回呼び出す（計6リクエスト）
4. レスポンスをフロント側でキャッシュ（sessionStorageなど）
5. 各日付にAPIデータ + 自前ロジックを合算してスコアを算出
6. スコア降順でランキング表示
```

## 8. 技術選定

| 用途 | 採用技術 | 選定理由 |
|---|---|---|
| フレームワーク | React + TypeScript | ロジックとUIの分離が容易。型安全で複雑なスコアロジックを管理しやすい |
| ビルドツール | Vite | 高速・シンプルで個人開発向き |
| 日付処理 | date-fns | 軽量・Tree-shaking対応・日本語ロケール対応 |
| UIコンポーネント | shadcn/ui | コピペで高品質UI、カスタマイズも容易 |
| スタイリング | Tailwind CSS | shadcn/uiと親和性が高く、レスポンシブ対応も容易 |
| 暦データ取得 | 暦API（koyomi.zingsystem.com） | 六曜・一粒万倍日・天赦日などが無料で一括取得可能 |
| デプロイ | Vercel | GitHubへのpushで自動デプロイ、無料枠で運用可能 |
